# Goal

Make `@opencode-ai/plugin` a runtime dependency of Supercode so OpenCode cache/package installs that omit development dependencies still include the plugin API package required during Supercode plugin load.

# Source Spec

- Approved spec: `docs/supercode/20260425-opencode-plugin-dep-b7c2/spec.md`
- Source-of-truth requirements:
  - Move `@opencode-ai/plugin` from root `devDependencies` to root `dependencies`.
  - Use a version range compatible with user-verified `@opencode-ai/plugin@1.14.24`, preferably `^1.14.24` unless repository constraints prove otherwise.
  - Update `bun.lock` consistently using the repository package manager.
  - Verify install/lockfile consistency plus existing test and typecheck commands.
  - Keep the change minimal and dependency-focused.

# Architecture / Design Strategy

This is a package metadata correction, not a code architecture change. Supercode imports `@opencode-ai/plugin` during plugin load/runtime, so the package must be declared under `dependencies` instead of `devDependencies`. The lockfile must be regenerated or updated by Bun to reflect the new dependency classification and compatible resolved version. No source implementation, plugin entrypoint, exports, package manager, or runtime fallback behavior should be changed unless verification proves the approved dependency-only fix cannot work.
